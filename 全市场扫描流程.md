## 📝 关键节点说明

### 1️⃣ **数据获取阶段**
- 调用 TDX API 获取股票列表
- 对每只股票获取150周的周线数据
- 计算技术指标（30周均线、成交量等）

### 2️⃣ **分析阶段**
- 判断股票所处阶段（1-4阶段）
- 只保留第二阶段（上升趋势）的股票
- 对第二阶段股票生成交易信号

### 3️⃣ **进度更新**
- 每处理完一批（10只）更新一次进度
- 前端每秒轮询一次获取最新进度
- 实时显示已处理数量、发现数量、耗时等

### 4️⃣ **结果保存**
- 扫描结果保存到SQLite数据库
- 交易信号添加到全局监测列表
- 可在"交易信号"页面查看

## ⏱️ **时间估算**

- **全市场股票数**: ~4500只（筛选后）
- **批次数**: ~450批（每批10只）
- **预计耗时**: 1-3分钟
  - 数据获取: ~30-60秒
  - 分析计算: ~30-90秒
  - 网络延迟: ~10-30秒

## 🔧 **技术实现要点**

### 后端实现
- **进度跟踪**: 全局变量 `scan_progress` 存储实时进度
- **进度回调**: `progress_callback` 函数在每批次处理后更新进度
- **并发控制**: 使用 `asyncio.gather()` 并发分析批次内的股票
- **批次处理**: 每批10只股票，批次间延迟0.5秒避免请求过快

### 前端实现
- **轮询机制**: 使用 `setInterval` 每秒查询一次进度
- **状态管理**: `scanProgress` 状态存储最新进度数据
- **UI更新**: 响应式更新进度条、批次进度、发现数量等
- **自动清理**: 扫描完成后清除轮询定时器

### API接口
- **扫描接口**: `POST /api/market/scan` - 启动扫描
- **进度接口**: `GET /api/market/scan/progress` - 查询进度
- **返回格式**: JSON格式，包含所有进度信息
